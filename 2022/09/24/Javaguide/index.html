基础概念与常识

  1. Java 语言特点

    1. 面向对象
    2. 跨平台
    3. 多线程
    4. 简单易学
    5. 安全可靠

  2. Jvm ， jdk， jre

    Jdk : java development kit

    ​ JVM 是运行 java 字节码的虚拟机。

    jre: java runtime environment

    jvm: java virtual machine

  3. 字节码

    .java - 编译 -> .class - 解释 -> 机器码

    Jvm 可以理解的代码就叫做字节码，由于字节码不针对特定机器，而是针对于 JVM，因此 Java 语言有跨平台特性。

  4. Java 语言“编译与解释并存”

    Java - 编译 -> 字节码 - 解释 -> 机器码

  5. openJDK vs Oracle jdk

    • Oracle jdk 大概每 6 个月发布一次主要版本， openJdk 大概每三个月发布一次。
    • OpenJdk 是一个参考模型并且是完全开源的，而 Oracle Jdk 是 OpenJdk 的一个实现，并不是完全开源的。
    • Oracle JDK 比 OpenJDK 更加稳定。
    • Oracle Jdk 比 OpenJDK 提供了更好的性能。

  6. Java vs C++

    同：都面向对象，支持封装，继承和多态。

    异：

    • Java 不提供指针以访问内存，内存安全。
    • Java 类是单继承的，但 C++ 支持多继承。
    • Java 有自动 GC。
    • C++ 支持方法重载和操作符重载，而 Java 只支持方法重载。

基本语法

  1. Java 注释

    单行，多行，文档

  2. 标识符 vs 关键字

    标识符就是一个名字，比如变量名，方法名等。

    而关键字是已经被 Java 赋予了特殊含义的标识符。

  3. continue、break 和 return

    continue：跳出当前循环，继续下一次的循环。

    break：跳出整个循环体。

    return：结束当前方法的运行。

  4. 成员变量与局部变量

    语法上：成员变量属于类，而局部变量实在代码块或方法中，成员变量可以被 public，private，static 等修饰符修饰。

    存储方式上： 对象存在与堆内存，局部变量存在于栈内存。

    生命时间：成员变量随着对象，局部变量随着方法。

    默认值：成员变量会被自动赋值，而局部变量不会。

  5. 静态变量

    静态变量是属于类的，可以被类的所有实例共享，无论创建多少对象，都共享同一份静态变量。

  6. 字符型常量和字符串常量

    形式：’s’，“string”

    含义：字符串常量相当于一个整型值，可参与表达式运算。字符串常量代表一个地址值。

    占内存大小：字符常量占2个字节，字符串常量占若干字节。

  7. 静态方法为什么不能调用非静态成员

    • 静态方法属于类，在类加载时会分配内存，而非静态成员属于实例对象，只有在对象实例化之后才存在，需要通过类的实例对象进行访问。
    • 类的非静态成员不存在的时候，静态成员可能就已经存在了，此时调用在内存中尚未存在的非静态成员，属非法操作。

  8. 静态方法和实例方法有何不同？

    1. 调用方式。
    2. 访问类成员是否存在限制。

  9. 重载和重写

    重载：相同方法名，不同参数列表

    重写：

    • 方法必须相同，访问修饰符大于或等于父类。

    • 父类方法被 private/final/static修饰则不能重写。

    • 构造方法不能重写。

    • 区别点 重载方法 重写方法
      发生范围 同一个类 子类
      参数列表 必须修改 一定不能修改
      返回类型 可修改 子类方法返回值类型应比父类方法返回值类型更小或相等
      异常 可修改 子类方法声明抛出的异常类应比父类方法声明抛出的异常类更小或相等；
      访问修饰符 可修改 一定不能做更严格的限制（可以降低限制）
      发生阶段 编译期 运行期

      方法的重写要遵循“两同两小一大”（以下内容摘录自，《疯狂 Java 讲义》 ）：

      • “两同”即方法名相同、形参列表相同；
      • “两小”指的是子类方法返回值类型应比父类方法返回值类型更小或相等，子类方法声明抛出的异常类应比父类方法声明抛出的异常类更小或相等；
      • “一大”指的是子类方法的访问权限应比父类方法的访问权限更大或相等。

      ⭐️ 关于 重写的返回值类型 这里需要额外多说明一下，上面的表述不太清晰准确：如果方法的返回类型是 void 和基本数据类型，则返回值重写时不可修改。但是如果方法的返回值是引用类型，重写时是可以返回该引用类型的子类的。

  10. 可变长参数

    注意：可变长参数只能作为函数的最后一个参数。

    调用方法时，若遇到方法重载，会优先匹配固定参数的方法。

    可变参数编译后实际会被转换成一个数组。

  1. 基本类型 vs 包装类型

    • 成员变量包装类默认为 NULL，基本数据类型不是。
    • 包装类可用于泛型，基本数据类型不可。
    • 基本数据类型的局部变量存放在 JVM 栈的局部变量表中，基本数据类型成员变量存放在堆中。而几乎所有对象实例都存在于堆中。
    • 基本数据类型占用的空间很小。

  2. 包装类型缓存机制

    Java 包装类型的大部分都用到了缓存机制来提升性能。

    Byte,Short,Integer,Long 这 4 种包装类默认创建了数值 [-128，127] 的相应类型的缓存数据，Character 创建了数值在 [0,127] 范围的缓存数据，Boolean 直接返回 True or False。

    创建一个包装类对象有两种方法：
    （1）构造器方法（就是new出来）；
    （2）自动装箱（就是编译器自动调用包装类的valueOf方法）；
    两种方法的区别：
    构造器方法：不论值的大小，返回的将都会是一个新对象；
    自动装箱会先经过判断，再决定返回的是一个新对象还是常量池中已存在的对象。

    当通过自动装箱机制创建包装类对象时，首先会判断数值是否在-128—-127的范围内，如果满足条件，则会从缓存（常量池）中寻找指定数值，若找到缓存，则不会新建对象，只是指向指定数值对应的包装类对象，否则，新建对象。

    在缓存值范围内的对象可以直接在常量池中取出，不用创建新的对象；
    当需要频繁的使用同一对象的时候，如果有缓存，则可以避免重复创建同一对象，节省空间开销和时间消耗，提升了性能。

  3. 自动拆装箱

    装箱：将基本数据类型用他们对应的引用类型包装起来。

    自动装箱时调用了包装类的 valueOf 方法。

    拆箱：将包装类型转换为基本数据类型。

    拆箱时调用了 xxxValue 方法

    频繁拆装箱会严重影响系统性能

  4. 为什么浮点数运算丢失精度？

    计算机在使用二进制表示一个数字时，宽度是有限的，无线循环小数存储在计算机时，只能被截断，就会导致小数精度损失。

  5. 如何解决浮点数运算的精度丢失问题？

    BigDecimal 可以实现对浮点数的运算，不会造成精度丢失。

  6. 超过 long 整型的数据该如何表示？

    在 Java 中，64 位 long 整形是最大的整数类型。

    BigInteger 内部使用 int[] 数组来存储任意大小的整形数据。

    但 BigInteger 运算效率较低。

面向对象基础

  1. 面向对象和面向过程的区别

    • 面向过程把解决问题的过程拆成一个个方法，通过一个个方法的执行解决问题。

    • 面向对象会先抽象出对象，然后用对象执行方法的方式解决问题。

      面向对象开发的程序一般更以维护，更容易，易拓展

  2. 类的构造方法的作用是什么

    构造方法是一种特殊的方法，主要工作是完成对象的初始化。

  3. 类的默认构造方法

    如果一个类没有声明构造方法，也可以执行。因为会自动生成一个默认的无参构造。但如果我们自己添加了构造方法（无论有参还是无参），Java 就不会再生成默认的无参构造方法。

  4. 构造方法有哪些特点？能否重写？能够重载？

    特点：

    • 方法名与类名相同。
    • 没有返回值，就算是 void 也不行。
    • 无需显式调用。

    构造方法不能被 Override。可以被 Overload。

  5. 面向对象的三大特征

    • 封装

      把一个对象的状态信息隐藏在对象内部。

    • 继承

      使用已存在的类的定义作为基础建立新类的技术。

      1. 子类拥有父类对象所有的属性和方法（包括私有属性和私有方法），但是父类中的私有属性和方法子类是无法访问，只是拥有。
      2. 子类可以拥有自己属性和方法，即子类可以对父类进行扩展。
      3. 子类可以用自己的方式实现父类的方法。

    • 多态

      一个对象具有多种的状态，具体表现为父类的引用指向子类的实例。

  6. 接口和抽象类的共同点和区别

    同

    • 都不能被实例化
    • 都可以包含抽象方法
    • 都可以有默认实现的方法（default）

    异

    • 接口主要用于对类的行为进行约束。抽象类主要用于代码复用。
    • 类只能单继承，而可以实现多个接口。
    • 接口中的成员变量只能是public static final类型的，不可被修改且必须有初始值。而抽象类的的成员变量默认为 default。

  7. 深拷贝和浅拷贝区别。什么是引用拷贝。

    • 浅拷贝会在堆上创建一个新对象（区别于引用拷贝），但如果原对象内部的属性是引用类型，浅拷贝会直接赋值内部对象的引用地址，新旧对象共用一个内部对象。
    • 深拷贝会递归地完全复制整个对象。
    • 引用拷贝使两个不同的引用指向同一个对象。

  2. ==和equals() 的区别

    • ==对于基本类型和引用类型的作用效果不同：
      • 对于基本数据类型来说，==比较的是值。
      • 对于引用类型来说，==比较的是对象的内存地址。
    • equals()不能用于判断基本数据类型，只能用于判断两个对象是否相等。
    • equals()方法存在两种使用情况：
      • 类没有重写 equals()方法：等价于使用==进行判断，使用 Object类equals()方法。
      • 重写equals方法：这个是由开发人员自己定的，一般是比较对象的属性是否相等。

    注意：String 中的 equals() 方法是已经经过重写的，所以字符串对象比较时可以直接调用String 中的 equals() 方法。

  3. hashCode() 有什么作用？

    hashCode() 的作用是用于获得哈希码（int 整数），哈希码的作用是确定该对象该哈希表中的索引位置。

    哈希表存储的是键值对，能根据键快速地索引出‘值’，这就需要用到哈希码。

    • 如果两个对象的 hashCode 值相等，那这两个对象不一定相等（碰撞）
    • 如果两个对象的 hashCode 值相等并且 equals() 方法也判断为 true，我们才认为这两个对象相等。
    • 如果两个对象的 hashCode 值不相等，我们直接就认为两个对象不相等。

  4. 为什么重新写 equals() 时必须重写 hashCode() 方法?

    equals 方法判断两个对象是相等的，那这两个对象的 hashCode 值也一定要相等。

    否则，equals 判断两个对象是相等的，而 hashCode 并不相等，会造成在hash表中不同索引上的 key 相同的情况。

String

  1. String、StringBuffer、StringBuilder 的区别:

    • 可变性
      • String 是不可变的
      • StringBuffer 和 StringBuilder 都继承自 AbstractStringBuilder 类，在 AbstractStringBuilder 中是用字符数组保存字符串，故可以修改字符串。
    • 线程安全性
      • String 对象是不可变的，是常量，线程安全。
      • StringBuffer 对方法增加了同步锁或对调用的方法加了同步锁，所以是线程安全的。
      • StingBuilder 并没有对方法加同步锁，所以是非线程安全的。
    • 性能
      • 每次对 String 类型引用进行改变时，都会产生一个新的 String 对象，然后引用指向新的 String 对象。
      • StringBuffer 是对本身进行操作，不会改变对象引用。
      • StringBuilder 线程不安全，性能比 StringBuffer 提高10% ~ 15%，但要冒线程不安全的风险。

    总结：

    • 操作少量数据：String
    • 单线程下操作大量数据：StringBuilder
    • 多线程操作大量数据：StringBuffer

  2. String 为什么是不可变的？

    • 保存字符串的数组被 final 修饰且为私有的，并且 String 类没有提供 / 暴露修改这个字符串的方法。（不要错误的以为该数组被 final 修饰就不能再更改，final 只是代表这个引用不能再指向其他对象，而不是说这个对象不能再进行更改）
    • String 类被 final 修饰导致其不能被继承，进而避免了子类破坏 String 不可变。

  3. 不是说 Java 不支持操作符重载吗，为什么为 String 重载了 + 和 +=?

    Java 本身并不支持操作符重载， + 和 +=是专门为 String 类重载过的运算符，也是 Java 中仅有的两个重载过的运算符。

  4. String 使用 + 拼接的原理

    实际是通过 StringBuilder 调用 append() 方法实现的，拼接成功后再调用 toString() 得到一个 String 对象。

    但在循环等情况下用 + 进行拼接的话，会有明显的缺陷：并不会复用创建的StringBuilder，会导致创建过多的 StringBuilder 对象。（因为 StringBuilder 是在循环内部被创建的）。

    所以这种情况下应当直接使用 StringBuilder 对象的 append() 方法进行字符串拼接。

  5. String#equals() 和 Object#equals() 有何区别?

    String 的 equals 方法是被重写过的，比较的是 String 字符串的值是相等。Object 的 equals 方法比较的是对象的内存地址。

  6. 字符串常量池

    字符串常量池是 JVM 为了提升性能和减少内存消耗，针对字符串专门开辟的一块区域，主要目的是避免字符串的重复创建。

  7. String#intern 方法有什么作用？

    String.intern() 是一个 native 方法，作用是将指定的字符串对象的引用保存在字符串常量池中，分为两种情况：

    • 如果字符串常量池中保存在对应的字符串对象的引用，就直接返回该引用。
    • 如果字符串常量池中没有保存对应的字符串对象的引用，那就在常量池中创建一个指向该字符串对象的引用并返回。

  8. String 类型的变量和常量做 + 时发生了什么：

    • 对于常量（String name = “a” + “b”）

      • 由于编译器的常量折叠会把常量表达式求出来的值作为常量嵌在最终生成的代码中，所以String name = "a" + "b" 会优化成 String name = "ab"

    • 对于变量

      • 加final（final String name = str1 + str2）

        实际上，加了 final 以后，编译器会将字符串变量当作常量来处理。因为编译器在程序编译期就可以确定它的值，其效果就相当于访问常量。

      • 不加 final（String name = str1 + str2）

        引用值在程序编译期是无法确定的，编译器无法对其进行优化。
