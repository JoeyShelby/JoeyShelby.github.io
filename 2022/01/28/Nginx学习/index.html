2. 基本概念

2.1 什么是Nginx

Nginx (engine x) 是一个高性能的HTTP和反向代理Web服务器，特点是占用内存少，并发能力强。

Nginx专为性能优化而开发，性能是其最重要的考量，实现上非常注重效率，能经受高负载的考验，有报告表明其能支持高达50,000个并发连接数。

2.2 反向代理

正向代理

​ 在客户端（浏览器）配置代理服务器，通过代理服务器进行互连网访问。（暴露的是最终服务器）

反向代理

​ 我们只需要将请求发送到反向代理服务器，由反向代理服务器去选择目标服务器获取数据后，再返回给客户端，此时反向代理服务器和目标服务器对外其实就是一股服务器，暴露的是代理服务器地址，隐藏里真实服务器IP地址。（暴露反向代理服务器，隐藏最终服务器）

总结

正向代理客户端，反向代理服务器

2.3 负载均衡

​ 我们可以增加服务器的数量，然后将请求分发到各个服务器上，将原先请求集中到单个服务器上的情况改为将请求分发到多个服务器上，将负载分发到不同的服务器。

2.4 动静分离

​ 为了加快网站的解析速度，可以把动态请求和静态请求由不同的服务器来解析，加快解析速度。降低原来单个服务器的压力。

3.3 Nginx的配置文件

Nginx配置文件位置

/usr/local/nginx/nginx.conf

Nginx配置文件内容

第一部分 - 全局块

​ 从配置文件开始到event块之间的内容，主要会设置一些影响Nginx服务器整体运行的配置指令。

​ 其中worker_processes 1;是Nginx服务器处理服务的关键配置，worker_processes的值越大，可以支持的并发处理量也就越多，但是受到硬件、软件等设备的制约。

第二部分 - events 块

​ events块涉及的指令主要影响Nginx服务器与用户的网络连接。

​ 其中worker_connections 1024;表示每个worker_process支持的最大连接数为1024。

​ 实际应用中，这部分对Nginx性能影响较大，应灵活配置。

第三部分 - http块

​ 这是Nginx服务器配置最频繁的部分。

​ 注意：http块也可以包括http 全局块， server 块。

http 全局块

​ 指令包括文件引入、MIME-TYPE 定义、日志自定义、连接超时时间以及单链接请求上限等。

server 块

​ 该块和虚拟主机有密切关系，虚拟主机总用户角度看，和一台独立的硬件主机是完全一样的，该技术的产生是为了节省互联网硬件成本。

5.3 master-worker机制的好处

  1. 可以使用nginx -s reload 实现热部署。

  2. 每个worker都是独立的进程，保证服务正常运行。

5.4 设置多少worker合理

​ 每个worker线程都可以把一个cpu的性能发挥到极致。所以worker数和服务器的cpu核心数相等是最为适宜的。设置少了会浪费cpu，多了会造成cpu频繁切换上下文带来损耗。

5.5 连接数worker_connection

  1. 发送请求时，占用的连接数为2个或4个。（2个为访问nginx静态内容，4个则代表还需访问tomcat服务器）

  2. 普通的静态访问的最大并发数为：worker_connections * worker_process/2 。二如果是HTTP作为反向代理来说，最大的并发数量应为worker_connection * worker_process/4。
