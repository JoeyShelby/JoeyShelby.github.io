SpringBoot 学习笔记

发表于 2022-06-19 分类于 学习 ， SpringBoot Valine：
SpringBoot 是在 Spring 基础上进一步封装的框架，在开发人员看来，省去了很多繁琐的配置，能更专注于业务代码开发。

一 入门

1. SpringBoot 是什么

Spring Boot makes it easy to create stand-alone, production-grade Spring based Applications that you can “just run”.

We take an opinionated view of the Spring platform and third-party libraries so you can get started with minimum fuss. Most Spring Boot applications need minimal Spring configuration.

SpringBoot 是在 Spring 的进一步简化，强调“约定大于配置”。与 SSM 项目相比，SpringBoot 项目所需要的配置文件是很少的，可以使得开发人员更专注于业务开发。

2. 微服务是什么

微服务架构风格是一种将一个单一应用程序开发为一组小型服务的方法，每个服务运行在自己的进程中，服务间通信采用轻量级通信机制(通常用HTTP资源API)。这些服务围绕业务能力构建并且可通过全自动部署机制独立部署。这些服务共用一个最小型的集中式的管理，服务可用不同的语言开发，使用不同的数据存储技术。

3. SpringBoot 项目生成

使用 IDEA 构建 SpringBoot 项目有两种推荐方式，一是从官网上生成项目结构，二是直接在IDEA中生成项目结构。

a) 使用官方网站提供的 initializr

地址 ：Spring Initializr

在该网页上选择 SpringBoot 开发相关配置信息，按下 GENERATE 后，便可以生成一个基本的 SpringBoot 项目结构。下载后导入到集成开发环境中即可。

b) 使用 IDEA 集成的 initializr

使用 IDEA 生成 SpringBoot 项目的原理与上面的方式是一样的。IDEA 只不过是集成了 initializr

选择好项目的相关配置信息之后，按下 next

选择需要的依赖，这里一般会选上 web 依赖。而后按下 create 即可创建一个最基础的 SpringBoot 项目。

3 run() 方法

  1. 推断应用的类型是普通项目还是 Web 项目。
  2. 查找并加载所有可用初始化器，设置到 Initializers 属性中。

4 @ConditionalOnXXX

在每一个 xxxAutoConfiguration 类上，都会有 @ConditionalOnXXX 注解，这些注解是在判断一些条件，如果注解所规定的条件在当前系统下并不满足，那么该自动配置类便不会生效。

三 配置文件

1 yaml 与 properties

使用 initializr 生成的项目默认的配置文件为 application.properties，而实际上更推荐使用 application.yaml。

  1. yaml 对空格，缩进要求严格。
  2. properties 只能存储键值对。
  3. yaml 不仅可以给 Map、数组以及实体类赋值，所以使用 yaml，可以很方便地进行全局配置。
  4. yaml 可以实现松散绑定。
  5. 使用 yaml 给实体类赋值时，可以很方便的实现 JSR303 校验。
  6. yaml 可以很好地结合 EL 表达式。

4 配置文件与配置类

在 org.springframework.boot.autoconfigure 包下，能发现很多的 xxxAutoConfiguration 类，这些 AutoConfiguration 类，都对应着一个 xxxProperties 类。很容易发现，这些 xxxAutoConfiguration 是根据 xxxProperties 实体的值进行配置的。而 xxxProperties 的属性值，便对应着配置文件中的配置信息。

3.3 使用

1 html 约束

xmlns 属性可以在文档中定义一个或多个可供选择的命名空间。该属性可以放置在文档内任何元素的开始标签中。该属性的值类似于 URL，它定义了一个命名空间，浏览器会将此命名空间用于该属性所在元素内的所有内容。

2 语法

在标签的原有属性前加上 th 便可使原有属性变成一个 thymeleaf 属性。如此便能做到 html 中所有元素都能被 thymeleaf 接管。

th: 属性

1）th:text：文本替换；

2）th:utext：支持html的文本替换。

3）th:value：属性赋值

4）th:each：遍历循环元素

5）th:if：判断条件，类似的还有th:unless，th:switch，th:case

6）th:insert：代码块引入，类似的还有th:replace，th:include，常用于公共代码块提取的场景

7）th:fragment：定义代码块，方便被th:insert引用

8）th:object：声明变量，一般和*{}一起配合使用，达到偷懒的效果。

9）th:attr：设置标签属性，多个属性可以用逗号分隔

If you want to keep those Spring Boot MVC customizations and make more MVC customizations (interceptors, formatters, view controllers, and other features), you can add your own @Configuration class of type WebMvcConfigurer but without @EnableWebMvc.

  • 不要添加 @EnableWebMvc 注解。（因 @ConditionalOnMissingBean({WebMvcConfigurationSupport.class})）
  • 通过重写方法达到扩展配置的目的。

4 手动切换语言

根据源码可知，SpringBoot 会根据请求头中的信息自动地配置 Locale。

想要手动切换语言配置的话，原理上对请求头中 Accept-Language 进行更改即可，方法有很多，比如通过 ajax 或在后端拦截到请求后根据请求参数对 Accept-Language 做出更改。

总之方法是很多的。

根据源码我们可以发现，SpringBoot 在自动装配时， @ConditionalOnMissingBean(name = {“localeResolver”}) 判断是否不存在名为 localeResolver 的 Bean，若存在，则该方法不起作用。

所以也可以自己写一个名为 localeResolver 的bean，并使其在容器中生效，这样就可以自己了处理国际化请求了，但这样的话需要在每次请求时都带有一个请求参数，也并不优雅。

属性 说明
driver-class-name 连接Mysql驱动类的全限定类名

url 数据库url，每一种数据库的url写法都是固定的，必须遵守
initial-size 初始化连接池连接数大小
min-idle 最小空闲连接数，一般等于initial-size
max-active 最大连接数
max-wait 获取连接时最大等待时间，单位毫秒
time-between-eviction-runs-millis 间隔多久检测需要关闭的空闲连接的时间，单位毫秒

validation-query 配置一个sql查询语句来检测连接是否有效
test-while-idle 设置申请连接的时候有空闲连接时进行检测，如果连接空闲时间大于timeBetweenEvictionRunsMillis，执行validationQuery检测连接是否有效。建议设置为true来保证安全性且不影响性能。
test-on-borrow 申请连接时执行validationQuery检测连接是否有效。设置为true时会影响性能
