在 C 语言中，数组名被解读为指向数组首元素的指针，也就是说，数组名标识一个地址。为什么将一个指针赋值给另一个指针会出现错误呢？

二 规则

  • 数组不能直接给数组赋值
  • 指针不能直接给数组赋值

三 原因

数组名是一个不可变的指针，不能对它进行加、减和赋值运算。也就是说，在 C 语言中，数组名是一个常量。

对数组进行赋值，实质上是在尝试把一个地址值赋值给一个常量指针，这当然是不被允许的。

四 解决办法

1 memcpy()

  • 作用：将地址从 src 开始的前 count 个字节的内容拷贝到地址从 dst 开始的内存空间中。

  • 返回值：目标空间的起始地址。

  • 需要注意：dst 和 src 在内存中的位置不能重叠，否则会产生错误。

2 memmove()

  • 作用：同 memcpy函数一样，但细节上有一点区别，该函数不需要注意 dst 与 src 在内存空间上的重叠问题。
  • 返回值：目标空间的起始地址。
  • 需要注意：相对于 memcpy 函数，该函数效率较慢。

3 memccpy()

  • 作用：由 src 所指内存区域复制不多于 count 个字节到 dest 所指内存区域，如果遇到字符 c 则停止复制。
  • 参数
    • dest：目的
    • src：源
    • c：复制停止字符
    • count：复制字节数
  • 返回值：如果 c 没有被复制，则返回 NULL ，否则，返回 dest 中字符 c 后面一个字符位置的地址。

4 strcpy()

  • 作用：拷贝字符串，把 src 所指向的字符串复制到 dest。
  • 返回值：该函数返回一个指向最终的目标字符串 dest 的指针。
  • 需要注意：与前三个函数不同，该函数只能用于字符数组的拷贝。
